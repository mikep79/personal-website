Word of the day: Fibonacci sequence

Named after an Italian mathematician, the Fibonacci sequence is a list of numbers starting with 1, where each number is the sum of the two previous numbers. (for example, 1, 1, 2, 3, 5, 8, etc.) This is a fun puzzle that's actually pretty simple to solve. Here's a solution that prints out just the 20'th number, and then prints the first 20 in rows:

var fibNumbs = new List < int > { 1 , 1 };

for ( int i = 0 ; i < 18 ; i ++) {

var currentIndex = fibNumbs . Count - 1 ;

var previousIndex = fibNumbs . Count - 2 ;

fibNumbs . Add ( fibNumbs [ currentIndex ] + fibNumbs [ previousIndex ]);

}

Console . WriteLine ( $"The { fibNumbs . Count }'th fib numb is: { fibNumbs [ fibNumbs . Count - 1 ]}." );

// print first 20 numbers:

foreach ( var numb in fibNumbs ){

Console . WriteLine ( numb );

}

Word of the day: Double Precision

"Refers to a type of floating point number (decimal) that has more precision (that is, more digits to the right of the decimal point) than a  single-precision number."

An interesting difference I'm learning between JavaScript and C# is the need to specify the data type for each variable you create, with detail. For example, in JavaScript we might use the variable "thing" to represent a number or text without having to specify which:

var thing = 'string of letters'; 

var thing = 3.14;

In C#  we must specify not only whether this is a number or a string, but also the type of number:

string thing = "string of letters";

int thing = 3;

double thing = 3.14;

When using numbers, we can use different levels of precision required (by using more digits). Double precision allows us to use more digits for our number, but each number used requires more data.

Word of the day

String interpolation:

Process of evaluating a string literal containing placeholders (variables) replaced with values.

C# example

string breakfastFood = "waffles";

Console.WriteLine($"Pancakes are almost as tasty as {breakfastFood}.");

$ is used before a string, as a "heads up" to the compiler that the following string contains some logic! Our variable is then inserted using the { }. When I was learning JavaScript, we pretty much just concatenated our strings using , or +. The one instance where I used string interpolation I can think of is with AngularJS expressions in an html file:

<p>Here's a {{ vm.word }} example of string interpolation.</p>
